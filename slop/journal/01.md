## Architecture Decisions

### Hexagonal Architecture (Ports & Adapters)
- **Core** contains domain models and trait interfaces (ports)
- **Providers** will implement `PaperProvider` trait (adapters)
- **Services** will implement service traits (business logic)
- **Transport layers** (Poem, MCP, CLI) wrap core without coupling

### Key Design Choices

1. **Generic `download_url`** instead of `pdf_url` - supports multiple formats
2. **Explicit imports** instead of wildcards - prevents naming conflicts
3. **Error categorization** - enables intelligent retry strategies without coupling
4. **`Send + Sync` bounds** - ensures thread-safe trait objects
5. **Optional trait methods** - reduces boilerplate for provider implementations

## Issues Resolved

1. **Name collision**: `core` conflicts with std library → renamed to `paper-fetch-core`
2. **Edition error**: Used non-existent "2024" → changed to "2021"
3. **Wildcard imports**: Caused ambiguity → switched to explicit imports
4. **Duplicate types**: `PaperFetchError` in two files → kept only in error.rs
5. **Typos**: `confidence_scopre` → `confidence_score`, `search_al` → `search_all`

## Key Learnings

- **Workspace setup** requires all members to exist before Cargo accepts the config
- **Semantic versioning**: Major.Minor.Patch (e.g., "1.0.228" not "1.0, 228")
- **async-trait** still needed for `dyn Trait` support even though Rust 1.75+ has native async traits
- **Pattern matching with guards** (`if e.is_timeout()`) enables refined error categorization
- **Transport-agnostic design** means zero HTTP/MCP dependencies in core library
